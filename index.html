HTML

  • The head element contains metadata about the page and is not usually visible to visitors
  • The body element contains the page content which will be shown to visitors
  • The meta tag is used to identify metadata such as the page character set and language
  • Pages can be divided up into smaller sections using the 'section', 'div' and 'span' tags
  • These sections can be classified using both a class and an id identifier
  • Lists can be created using the ul (unordered list) and ol (ordered list) tags
  • List items are identified using the 'li' tag
  • Headers are identified with the letter H and a number from 1 to 6, e.g., H1
  • Paragraphs are identified using the p tag

CSS

  • CSS elements begin with a reference to the selector (a tag, class or id), followed by the declaration containing the property and then the attribute value.
  • CSS can be presented in three ways. Through inline CSS, internal CSS (in a dedicated area of the HTML code) or external CSS using a separate document.
  • Marin indicates how much space we want around the outside of an element.
  • Padding indicates how much space we want around the content inside an element.
  • Classes can be referenced by placing a full stop in front of the element (e.g., .card)
  • IDs can be referenced by placing a hash in front of the element (e.g., #css)

Git

  • git status -- check which branch you are on and any uncommited changes to the files in that branch
  • git checkout -b branch-name -- creates a new branch and switches to it
  • git add -- adds files to the staging area
  • git commit -m "message" -- commits changes in the staging area
  • git push -- pushes changes to the repository
